6.3 Critical Webflow E-commerce Limitations

Unsupported Features in Webflow

The following features are NOT supported in Webflow and cannot be implemented natively:

  • Saved Addresses: No ability for customers to save multiple shipping addresses for future use
  • Address Book: Cannot implement an address management system for customers
  • Payment Options: Limited to only Stripe and PayPal (no bank transfer, cash-on-delivery, or other payment methods)
  • Custom Checkout Fields: Highly restricted - only three additional fields possible (one text field, one text area, one checkbox)
  • Wishlist/Favorites: No native support for saving favorite products
  • User Authentication: No social login options (Google, Facebook, etc.)
  • Order History: No customer login or order history tracking
  • Complex Discount Rules: Limited discount functionality (no volume discounts, tiered pricing)
  • Shipping Zones: No support for zip code-based shipping, only country-based

These limitations mean several screens from the original design cannot be implemented on Webflow and will need to be excluded or significantly simplified.

6.4 Webflow E-commerce Management

Webflow provides a comprehensive e-commerce management system with these native features:

  • Product Collections: Categories collection plus unlimited custom collections for filtering
  • Orders Management: Built-in order processing and fulfillment tracking
  • Discount Management: Create and manage promotional codes
  • Subscription Management: Handle recurring billing (with limitations)
  • Custom Fields: Add unlimited custom fields to products

6.5 Webflow Product Structure

Webflow offers a flexible product structure that can be customized for your needs:

Basic Product Structure

  • Name: Product title
  • SKU: Unique product identifier
  • Price: Base product price
  • Compare-at Price: Original price for sale items
  • Category: Primary product category
  • Description: Rich text product description
  • Images: Product gallery images
  • Variations: Size, color, and other options
  • Weight: For shipping calculations
  • Dimensions: Physical product dimensions

You can add as many custom fields as needed for specific requirements, such as material type, care instructions, warranty information, or any other product attributes.

6.6 Required E-commerce Pages

Webflow automatically generates these core e-commerce pages, which can be fully styled to match your design:

  • Products: Main catalog page with filtering and sorting options
  • Categories: Collection pages for product organization
  • Discounts: Management interface for promotional offers
  • Orders: Administrative dashboard for tracking customer purchases
  • Subscriptions: Management interface for recurring billing

All of these pages can be styled according to your needs while maintaining their core functionality. However, no additional features beyond what Webflow offers natively can be integrated without custom code solutions, and even with custom code, many features remain impossible to implement due to Webflow's closed checkout system.

6.9 Webflow Limitations vs. Custom Solution

Limited API Access

Webflow provides extremely limited API access compared to a fully custom solution, restricting most integrations and custom functionality.

Subscription Limitations

Basic subscription functionality with limited flexibility for complex recurring billing models.

Complex Discount Structures

Limited ability to implement sophisticated discount rules or loyalty programs.

Authentication Options

No support for social authentication, custom authentication flows, or user account management.

Checkout Customization

While stylable, the checkout process has severe structural limitations that cannot be customized beyond basic styling - no saved addresses, limited payment methods, and minimal custom fields.

7. Recommended Focus Areas

Selecting the right implementation approach depends on your specific needs:

Choose Medusa.js + Next.js When...

  • You need complex custom features not supported by Webflow
  • Social authentication is a core requirement
  • You plan to scale significantly with custom integrations
  • Advanced product variations or inventory management are needed
  • You require complete control over the checkout process
  • Original design requires saved addresses, order history, or advanced payment options
  • Custom checkout flow with multiple steps is required

Choose Webflow E-commerce When...

  • You need to launch quickly (4-6 weeks vs 8-10 weeks)
  • Budget is more constrained (€4,500 vs $9,000)
  • Content management by non-technical staff is important
  • You're willing to simplify the design to work within Webflow's limitations
  • Visual design flexibility is prioritized over backend complexity
  • Long-term maintenance simplicity is valued
  • You accept that certain screens and functions must be excluded (saved addresses, multiple payment methods)

7.1 Implementation Reality Check

Adaptations Required for Webflow Implementation

If choosing the Webflow option, these adaptations will be necessary:

Screens That Must Be Eliminated:
  • Saved addresses management screens
  • Order history and detailed tracking screens
  • Wishlist/favorites functionality
  • User account management beyond basic information
  • Advanced payment option selections
Features That Must Be Simplified:
  • Checkout flow limited to Webflow's standard structure
  • Product filtering options limited to basic categories
  • Shipping options reduced to country-based options only
  • Order confirmation and notifications using standard templates

The Medusa.js option can fully implement the original design with all planned features intact.

To ensure successful implementation with either approach:

1. Early Infrastructure Setup

  • Complete hosting configuration in first week
  • Ensure development environments match production

2. Prioritize Core User Journey

  • Focus on product browsing → cart → checkout flow first
  • Ensure payment methods work flawlessly

3. Parallel Development Tracks

  • Frontend styling can progress alongside backend integration
  • Custom features can be developed in parallel with core pages

4. Weekly Progress Milestones

  • Set specific completion targets for each feature
  • Regular testing of completed components
